Vault ‚Äî Node.js + MongoDB Login System
A production-ready authentication system with JWT tokens, bcrypt password hashing, rate limiting, and account lockout protection.
Features

Registration with input validation (name, email, strong password)
Login with JWT token response
Account lockout after 5 failed attempts (2-hour lock)
Password hashing with bcrypt (12 salt rounds)
JWT authentication middleware for protected routes
Role-based access control (user / admin)
Rate limiting ‚Äî global (100/15min) + auth endpoint (10/15min)
Security headers via Helmet.js
Input validation via express-validator
Profile update and password change endpoints
Polished frontend UI with password strength meter

Quick Start
1. Install Dependencies
bashnpm install
2. Configure Environment
bashcp .env.example .env
# Edit .env with your MongoDB URI and a strong JWT_SECRET
3. Start MongoDB
bash# Local MongoDB
mongod

# Or use MongoDB Atlas ‚Äî paste your connection string in .env
4. Run the Server
bash# Development (with auto-restart)

# Production

Server runs at http://localhost:3000

Password Requirements

Minimum 8 characters
At least one uppercase letter (A-Z)
At least one lowercase letter (a-z)
At least one number (0-9)

Security Details

Passwords hashed with bcrypt (12 rounds)
JWT tokens expire in 7 days (configurable)
Auth endpoints rate-limited to 10 requests/15 min
Account locked for 2 hours after 5 failed login attempts
password field never returned in API responses (select: false)
HTTP security headers via Helmet
